# 浏览器书签目标回归测试清单

用于验证 `BrowserBookmarkSelector.validateTargetInCurrentBrowser` 在 quickSave 中对浏览器书签保存目标的修正逻辑，覆盖书签目标、目录目标、路径漂移以及 `placement` 保留规则。

## 测试说明

- 默认入口：`quickSave`
- 默认前置：`display.browserBookmarkSave.mode = browser`
- 除非用例另有说明，验证对象都是“重新打开 quickSave 后读取并修正已保存的 preference”
- “默认设置”指当前代码中的默认浏览器目标，即书签栏目录底部

## 书签目标

### 1. 书签目标存在，目录和路径都未变化

前置条件：

- 已保存一个书签目标
- 目标书签仍存在
- `nodeId` 对应书签的 `url` 与记录一致
- 目标书签仍位于记录中的 `folderId`
- 目标目录的实际路径与记录的 `pathIds` 一致

操作步骤：

1. 打开 quickSave
2. 等待浏览器书签目标初始化完成

预期结果：

- 目标仍是原书签目标
- `nodeId`、`folderId`、`kind=bookmark` 保持不变
- 原 `placement` 保留
- quickSave 中显示的目标目录名称正确

### 2. 书签目标不存在，原目录仍存在且未变化

前置条件：

- 已保存一个书签目标
- 删除该目标书签
- 原 `normalized.folderId` 对应目录仍存在
- 目录名称未变化
- 目录路径与记录的 `pathIds.slice(0, -1)` 一致

操作步骤：

1. 打开 quickSave

预期结果：

- 目标从书签回退为目录目标
- 返回的目标 `kind=folder`
- 目录为原书签所在目录
- `placement` 使用目录默认值，不保留原书签的 `before/after`

### 3. 书签目标不存在，原目录也不存在

前置条件：

- 已保存一个书签目标
- 删除目标书签
- 同时删除其原所在目录，或将目录改名使其视为不存在

操作步骤：

1. 打开 quickSave

预期结果：

- 目标校验失败
- quickSave 回退到默认设置
- 默认目标为书签栏目录底部

### 4. 书签目标仍存在，但被移动到其他目录

前置条件：

- 已保存一个书签目标
- 目标书签仍存在且 `url` 一致
- 将该书签移动到另一个目录

操作步骤：

1. 打开 quickSave

预期结果：

- 目标从书签回退为该书签当前所在的目录目标
- 返回的目标 `kind=folder`
- `folderId` 为书签当前实际所在目录
- `pathIds`、`pathTitles` 更新为该目录当前路径
- `placement` 使用目录默认值

### 5. 书签目标仍存在，未移动，但上层目录路径发生变化

前置条件：

- 已保存一个书签目标
- 目标书签仍存在且 `url` 一致
- 目标书签仍位于记录中的 `folderId`
- 调整目标目录的上层结构，使目录路径变化但目录本身不改名

操作步骤：

1. 打开 quickSave

预期结果：

- 目标仍是书签目标
- `kind=bookmark`、`nodeId` 保持不变
- `pathIds`、`pathTitles` 更新为目录当前实际路径
- `placement` 不保留，回到书签默认值

### 6. 书签目标仍存在，但 `url` 已变化

前置条件：

- 已保存一个书签目标
- 目标 `nodeId` 仍存在
- 修改该节点 URL，使其与记录中的 `url` 不一致
- 原目录仍存在

操作步骤：

1. 打开 quickSave

预期结果：

- 该书签视为“不存在”
- 按“书签不存在”逻辑回退到原目录目标
- 不应继续把该节点当成有效书签目标

### 7. 历史数据缺少书签目标的 `url`

前置条件：

- 已保存一个历史书签目标
- 记录中无 `url`
- 目标书签 `nodeId` 仍存在
- 书签目录与路径均未变化

操作步骤：

1. 打开 quickSave

预期结果：

- 兼容旧数据，目标仍可用
- 目标仍是书签目标
- 原 `placement` 保留

## Placement 规则

### 13. 书签目标未发生任何变化时保留 `placement`

前置条件：

- 已保存书签目标，`placement=before`
- 书签、目录、路径均未变化

操作步骤：

1. 打开 quickSave

预期结果：

- 目标仍为该书签
- `placement` 仍为 `before`

### 14. 书签目标发生任一修正时重置 `placement`

前置条件：

- 已保存书签目标，`placement=before`
- 分别验证以下场景：
  1. 书签不存在但目录仍存在
  2. 书签移动到其他目录
  3. 目录路径发生变化

操作步骤：

1. 对每个场景分别打开 quickSave

预期结果：

- 场景 1 和 2 中，目标变为目录且 `placement` 为目录默认值
- 场景 3 中，目标仍是书签但 `placement` 为书签默认值

### 15. 目录目标未发生变化时保留 `placement`

前置条件：

- 已保存目录目标，`placement=top`
- 目录名称和路径均未变化

操作步骤：

1. 打开 quickSave

预期结果：

- `placement` 仍为 `top`

### 16. 目录目标发生修正时重置 `placement`

前置条件：

- 已保存目录目标，`placement=top`
- 目录名称未变化，但目录路径发生变化

操作步骤：

1. 打开 quickSave

预期结果：

- 目标仍为目录
- `placement` 回到目录默认值

## 集成验证

### 17. 修正后的目标被写回选择器状态

前置条件：

- 构造一个“目标仍可修复但已不完全匹配”的场景
- 例如书签被移动，或目录路径发生变化

操作步骤：

1. 打开 quickSave
2. 观察浏览器书签目标选择器的展示
3. 直接保存当前页面
4. 关闭并重新打开 quickSave

预期结果：

- 第一次打开时，选择器展示的是修正后的可用目标
- 保存后不会再回退到旧的无效路径
- 第二次打开时读取到的仍是修正后的目标

### 18. 编辑已有 Chrome 书签时锁定目标不受历史 preference 干扰

前置条件：

- 当前页面对应一个已存在的 Chrome 书签
- 同时准备一份无效或过期的 `display.browserBookmarkSave` preference

操作步骤：

1. 打开 quickSave 进入编辑态

预期结果：

- 选择器锁定到当前编辑书签的实际目标
- 锁定目标通过当前书签实时构造
- 不受历史 preference 中旧 target 的影响
