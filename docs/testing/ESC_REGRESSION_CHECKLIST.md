# ESC 回归测试清单

用于验证 `Escape` 在 popup、quickSearch 以及多层弹窗场景下的行为优先级，避免一次按键同时退出多个状态。

## Popup

### 1. 搜索模式 + 批量编辑

1. 打开 popup
2. 进入搜索模式
3. 执行一次搜索，得到搜索结果
4. 在搜索结果中进入批量编辑模式
5. 按 `ESC`

期望结果：

- 只退出批量编辑模式
- 仍停留在搜索模式
- 搜索框内容保留
- 搜索结果保留

### 2. 仅搜索模式

1. 打开 popup
2. 进入搜索模式
3. 不进入批量编辑模式
4. 按 `ESC`

期望结果：

- 退出搜索模式

### 3. 搜索模式 + 设置弹窗

1. 打开 popup
2. 进入搜索模式
3. 打开设置弹窗
4. 按 `ESC`

期望结果：

- 只关闭设置弹窗
- 不能一次同时关闭设置弹窗和搜索模式

### 4. 标题编辑态

1. 打开 popup
2. 进入书签标题可编辑态
3. 按 `ESC`

期望结果：

- 只取消当前标题编辑
- 恢复原标题
- 不应顺带关闭搜索模式、设置弹窗或其他外层状态

## QuickSearch

### 5. 搜索结果 + 批量编辑

1. 打开 quickSearch
2. 输入关键词，得到搜索结果
3. 进入批量编辑模式
4. 按 `ESC`

期望结果：

- 只退出批量编辑模式
- 搜索词保留
- 结果列表保留
- quickSearch 窗口不关闭

### 6. 有搜索词但未进入批量编辑

1. 打开 quickSearch
2. 输入关键词
3. 不进入批量编辑模式
4. 按 `ESC`

期望结果：

- 按当前既有设计执行搜索输入框的 `ESC` 行为
- 如果输入框有内容或历史面板打开，则清空搜索或隐藏历史
- 不应误触发批量编辑退出逻辑

### 7. 连续按两次 ESC

1. 打开 quickSearch
2. 输入关键词并得到搜索结果
3. 进入批量编辑模式
4. 第一次按 `ESC`
5. 第二次按 `ESC`

期望结果：

- 第一次只退出批量编辑模式
- 第二次才执行 quickSearch 原本的搜索框 `ESC` 行为

### 8. 重命名弹窗

1. 打开 quickSearch
2. 拉起重命名弹窗
3. 按 `ESC`

期望结果：

- 只关闭重命名弹窗
- 不影响批量编辑、搜索词、结果列表或窗口状态

### 9. 编辑标签弹窗

1. 打开 quickSearch
2. 拉起编辑标签弹窗
3. 按 `ESC`

期望结果：

- 只关闭编辑标签弹窗
- 不影响下层状态

### 10. 确认弹窗

1. 打开 quickSearch
2. 拉起确认弹窗
3. 按 `ESC`

期望结果：

- 只关闭确认弹窗
- 不影响下层状态

## 多层优先级

### 11. 多层状态逐层退出

1. 打开 quickSearch
2. 输入搜索词并进入批量编辑模式
3. 在此基础上再打开确认弹窗或其他模态框
4. 连续按 `ESC`

期望结果：

- 按层级逐层退出
- 先关闭最上层弹窗
- 再退出批量编辑模式
- 最后才执行搜索框自己的 `ESC` 行为

## 边界场景

### 12. 退出编辑后选择状态重置

1. 在 popup 或 quickSearch 中进入批量编辑
2. 选择多条记录
3. 按 `ESC`

期望结果：

- 退出编辑后选择数归零
- 复选框恢复未选中
- 列表项取消选中高亮

### 13. 重复进入/退出编辑模式

1. 进入批量编辑模式
2. 按 `ESC` 退出
3. 再次进入批量编辑模式
4. 再按 `ESC`

期望结果：

- 两次行为一致
- 不应出现第一次正常、第二次异常的情况

### 14. 鼠标 + 键盘混合操作

1. 先搜索
2. 再多选
3. 再打开弹窗
4. 最后按 `ESC`

期望结果：

- 不会一次退出多个层级
- 当前最上层状态始终优先响应
